The Core features

  1. Import of all Amazon (FBA/FBM) orders from every Amazon Marketplace
    Based on order date. Also the cron job can automate the process. Set debitor, sales team, price list per Marketplace.
  2. Log your data
    Get every detail of the imported order and also submitted feed. Get to know the feed result for any case you have.
  3. FBM: Sync stock for all or single product
    You can sync the stock manually for a single product or all products by marketplace or run the Stock sync by cron. Set WH and location per Marketplace.
  4. FBM: Update order status
    Once you have shipped our orders the connector will set the tracking reference and the status to shipped on Amazon.
  5. FBA: Set fiscal positions with Taxes for all cross border situations (OSS compatible)
    As it is out of your influence from where Amazon is shipping the orders, it can be necessary to set taxes and income accounts per country. With the connector you can set any possible way of shipping from to country. Also OSS (for Europe) is covered.
  6. FBA: Split orders into different shipments and invoices per WH (VAT relevant)
    This topic is related to VAT as well. When a customer orders 2 products Amazon is able to ship it from different WH and countries. Based on fiscal position the VAT and income account will be set for the correct invoice.
  7. FBA: Get the FBA stock into Odoo
    Run by cron a stock report to have all product stock in Odoo. Offer also your FBA stock to any other Marketplace or Shop.
  8. FBA: Get and process FBA returns automatically (Helpdesk)
    The core feature from Odoo native is already good and we did build the process on it. When running the return report there will be tickets created with a return and refund from existing shipment. Moreover you will get an overview of returned reason from the report for statistical purpose.
  9. FBA: Upload your Odoo invoices into Amazon
    Amazon is forcing you to upload your invoice if you sell FBM or FBA by creating your own invoices.
  10. FBA: Run the VIDR report for B2B VAT invoices und uploads
    If you create your own invoices but use FBA than you are forced to upload the invoices within 24hours after shipment. For that reason we have implemented the VIDR report to check for shipments and upload the invoice by cron.
  11. Create outbound shipments from Odoo
    If you sell on other marketplaces or shops but want to use also your FBA stock you can do so by shipping your FBA inventory to customers. These "outbound shipments" can be created from the sale order.
