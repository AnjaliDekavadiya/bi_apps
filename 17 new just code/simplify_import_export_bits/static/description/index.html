Import Export

These access management export and import features provide a flawless process for transferring rules from the
staging environment to the production environment. Users can use the export feature to retrieve their created
access rules and save configurations for later use. Meanwhile, the import feature allowed previously exported
files to be reintegrated into the production environment. When these features are used together, they improve
control and efficiency in managing access control configurations across multiple environments.

Export

Export the file from access management

The Export feature selectively allows you to export the rules you've created in access management. The
export feature makes it easier to move access management rules from staging to production. Take note that
files will be exported in the form of JSON.

In order to export, first select the records you want to export, then go to the Action button and click on the
'Export File' button, as shown in the screenshot above.

Import

Import the file into access management

The Import feature allows you to transition from staging to production by allowing you to easily import the previously exported files. In order to import, click on the import icon as shown in the screenshot. Upload the file using the Upload Your File button on the new form.

After uploading the file, you can test it with the Test button given in the loft top corner
and you can see below screen
  • If you enable the Import Tag option, the import tag will appear alongside the rules.
  • If you want to import rules that are currently active, disable the Active button.
  • If there is any missing data when you import the rules, it will be defined in the Missing Data field,
     as shown in the screenshot, so make sure you have all the needed data.
  • Click on the Import button to import the rules in access management.

Why would you use this features?

These features make it easier to transfer access rules between staging and production environments, ensuring that security configurations are consistent. The export function saves created rules for later use, while the import function reintegrates them into production, thereby simplifying access management.

What are the access management export and import features, and how do they work?

The export feature retrieves and saves user-defined access rules as files. The import feature then makes use of these files to recreate the rules in a new environment. When used in sync, they ensure that access controls are consistent.

How do these features improve control and efficiency in access management?

These features improve control and efficiency by providing a reliable and standardized method for transferring access control configurations. They reduce the risk of errors and ensure that access rules are consistent across different environments.

Can you provide an example of how the export and import features can be beneficial?

Assume you're working on a software development project. Access rules are exported from the testing environment and imported into the live production environment. This ensures that the same security settings are used, saving time and reducing the possibility of misconfiguration.
