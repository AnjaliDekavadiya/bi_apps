Advance Product Dimension and Measurement(Height-Width-Length) Odoo App

Advance Product Measurement and Product Dimension(Height/Width/ Length)Odoo Apps

Product Dimension (Height/Width/Length) Value on Sales, Purchase, Invoice and Manufacturing

Advance Product Measurement & Dimension Odoo app helps you to add product measurement variable height, width and Length on product, based on that it will dynamically calculate Quantity of Product in Meter Square/Meter Cube and Price of it. This plugin will be mainly useful for Banner and Paper industry , where customer can order product in different dimension with different height, width, Length and based on that price need be calculated for that product size.
Also if you are not managing the stock of your product and purchasing it from any other supplier . then with default odoo procurement method "Make to order + buy" will generate Purchased same as requested from the customer.
Same if you are manufacture and you make product based on customer request for different size this plugin also works for MRP and generate manufacturing(production) order for requested dimension(height/width/Length) by customer from the Sale.

This odoo app helps users to calculate product price based on dimension(M2/M3) and based on quantity. Users can configure minimum and maximum length, height, and width for the product. Users have the option to select "Dimension Method" and enter measurement for selected product for sales order, purchase order, customer invoice, vendor bill and manufacturing order.

Features

Product Variant Dimensional Values

User can select UOM and add different height, width and length.

Product Price Calculation Option

User have option to calculate product price based on product dimension or qty.

Dimension Method Option

User have option to select different product measurement option as "Length x Width x Height" or "Width x Height"

Calculates Product UOM (Mt.)2 based on Length, Height & Width.

Automatically calculates product UOM (Mt.)2 based on height and width, also price will be calculated based on (Mt.)2.

Product Variant Measurement

Under product variant user need to enable "Uses Dimensional Values" option and select "UOM Prompt" and enter minimum and maximum "Length, Width, and Height" for product variant.

Option 1 : Price Calculation based on Dimension(m2/m3)

Unit Price Calculation Configuration

User have to select "Price Calculation based on Dimension(m2/m3)" option under Sales > Configuration > Settings menu.

In this option net price will calculated based on : UNIT PRICE * QTY * M2.

Product Dimension on Sales Order

While creating quotation, salesman can select "Dimension Method" and add "Length, Width and Height" of product and based on that "Dim Qty, and Weight Ea." will automatically calculated in order line.

User can see the length height and width and net price of product is display on generated order.

Product Dimension on Customer Invoice

While invoice created from the sales order same height(Mt.)/width(Mt.) will be passed on invoice line for product. Also billing user can add length, height and width of product in invoice line.

Product Dimension on Purchase Order

When purchase generated by the purchase user, purchase user can add length, height and width of product in order line and based on that price will be calculated according to its height and width. Also same will be automatically effected when purchase has been generated from the sales order(by confirm sales).

User can see the height(Mt.), width(Mt.), (Mt.)2 and net price of product is display on purchase order and also for request for quotation.

Product Dimension on Vendor Bill

While supplier invoice created from the purchase order same dimension method, height(Mt.)/width(Mt.) will be passed on invoice line for product.

User can see the height(Mt.), width(Mt.), (Mt.)2 and net price of product is display on generated bill.

Product Dimension on Manufacturing

When Manufacturing order created by the production user can select dimension method, production user can add height(Mt.) and width(Mt.) of main product on manufacturing order and based on that (Mt.)2 price will be calculated according to its height and width. Also Same will be automatically effected when manufacturing has been generated from the sales order(by confirm sales).

User can see the height(Mt.), width(Mt.), (Mt.)2 and net price of product is display on Production Order.

Option 2 : Price Calculation based on Qty

Configuration for Calculation of Unit Price

User have to select "Price Calculation based on Qty" option under Sales > Configuration > Settings menu.

Warning/Validation

While user enter length which exceeding minimum or maximum length value, Warning raised.

While user enter width which exceeding minimum or maximum width value, Warning raised.

While user enter heigth which exceeding minimum or maximum heigth value, Warning raised.
